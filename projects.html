Digital Camera

I created my own digital camera using a webcam, battery pack, keypad, and a Raspberry Pi 3B+. I used fswebcam and various libraries to create a script that captures images in multiple modes: retro pixelated, outdoor, and standard. The keypad controls the camera interface, and the webcam flash signals when the photo process is complete. A battery pack powers the entire device, and a systemd service ensures the Python script launches automatically on boot.

Docketing Tool

During my internship, one of my tasks was to automate the docketing process. Docketing involves documenting time spent on tasks and formatting it into an Excel file. I built a Python tool that integrates Gmail search, keyword filtering, and date ranges. The script parses emails, extracts relevant data, and exports a neatly formatted Excel file. It also includes a standalone .exe version requiring no setup from the end user.

Home Server

I run a home server on an ACEPC T11 Mini PC with Ubuntu 24.04.2 LTS, managed entirely through SSH. Using Docker and Portainer, I set up an ad blocker, VPN, and password manager. I also previously ran a similar setup on a Raspberry Pi 3B+, but later dedicated the Pi to my digital camera project. My next step is creating a local backup system that automatically saves phone photos to an SSD.

Handheld Tetris Console

I built a miniature handheld Tetris console using a microcontroller, tactile pushbuttons, and a 0.96" OLED display. The entire game runs on embedded C/Python (depending on board), including piece rotation, collision logic, score tracking, and rendering. This project taught me a lot about display protocols, button debouncing, microcontroller timing loops, and writing optimized game logic for small hardware.

(Hack Western 12 Submission)

Inspiration:

ClearHear was inspired by the ongoing struggles faced by my best friend who is hard of hearing. In class, she was unable to sufficiently focus because she had to take notes, read the board, and read the professor's lips all at once. It was jarring to observe the struggle for something I had taken for granted to be easy. I wanted to do something about it to address this injustice.

What it does:

ClearHear takes audio files and outputs raw transcripts along with accessible notes using Gemini AI, so that students who are hard of hearing or who face other disabilities can learn alongside their peers without having to juggle multiple invisible tasks just to keep up.

How I built it:

I built ClearHear as a Streamlit web application using Python and Google Gemini. Gemini performs transcription, tone indication, quotes from the speaker with timestamps, and structured note generation. I created a text-based output schema using delimiter markers so that the formatting remains consistent. The notes are made visually accessible using highlights, boldening, colour-coded tone indicators, and key concept emphasis. Finally, users can download the output as either a .txt or .docx file.
